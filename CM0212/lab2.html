Implement in Java a flood fill algorithm (similar to the facilities available in standard graphics packages such as Photoshop) using a stack (rather than recursion).

Take as input a binary PGM image in which values are either 0 or 255; this can be stored as a two-dimensional array containing integer pixel values. Select a position in the image (either hardcoding it or taking input from the user) and fill it and all neighbouring white (255) pixels, recolouring those pixels gray (e.g. 128). Continue this process until none of the pixels in the gray region have white neighbouring pixels. Remember to check for image boundaries to avoid "falling off" the image array.

So you will have to check a pixel's neighbours, its white neighbours' neighbours, their neighbours, and so on. To keep track of all these neighbours the stack is used to store them temporarily. Neighbours waiting to be processed are pushed on the stack and then popped off later when your algorithm is ready to processed them.
